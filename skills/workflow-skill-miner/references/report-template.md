For each top candidate, include:

- Trigger examples: realistic user requests that should activate the skill.
- Proposed frontmatter description: one concise draft.
- Workflow summary: 3-6 steps.
- Resource plan: files or scripts to include.
- Validation plan: how to test the skill on realistic tasks.
- Risks: privacy, security, maintenance, or overfitting concerns.

## Do Not Package Yet

List ideas with weak evidence, unclear triggers, too much private context, or insufficient recurrence. State what evidence would change the decision.

## Immediate Next Steps

List 1-3 actions, such as:

- Create the highest-ranked skill.
- Collect two more examples for a medium-confidence candidate.
- Add a small script or reference to an existing skill instead of creating a new one.
